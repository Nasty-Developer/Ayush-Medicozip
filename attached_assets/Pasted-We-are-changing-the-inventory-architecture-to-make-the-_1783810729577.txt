We are changing the inventory architecture to make the website scalable and production-ready.

The current Firestore-based medicine catalogue is not suitable for 50,000+ medicines because of Firestore quota limits, write limits, aggregation limits, and performance issues.

DO NOT delete any existing code. Preserve the current UI, routes, components, authentication, orders, and admin panel.

====================================================
NEW ARCHITECTURE
====================================================

Move the medicine catalogue from Firestore to PostgreSQL.

Firestore will only be used for real-time application data.

====================================================
IMPORTER
====================================================

Build a production-ready importer.

Import these files:

PRODUCT.SDF
STOCK.SDF
COMPANY.SDF
CATEGORY.SDF
DRUG.SDF

The importer must:

• Parse every file
• Merge relationships correctly
• Prevent duplicates
• Update existing medicines
• Insert new medicines
• Update stock
• Update companies
• Update categories
• Update drug groups

Everything should happen inside PostgreSQL.

Do NOT import medicines into Firestore anymore.
