This is a Replit platform limitation, not something in your project's code — so I can't fix it directly, but here's what's going on and how to work around it on Android:

Why it fails: Downloading a ZIP requires the mobile browser to handle a large binary file download. Android browsers (especially Chrome) often block this if they treat it as a pop-up, run out of memory generating the archive, or lack storage permissions granted to the browser.

Ways to get your files on Android:

Use the Replit mobile app instead of the browser — it handles file downloads more reliably than mobile Chrome.
Request "Desktop site" in your Android browser (Chrome menu → "Desktop site") before downloading — this sometimes changes how the download is triggered and avoids the pop-up block.
Check storage permissions — make sure your browser app has storage/file access permission in Android Settings → Apps → Chrome → Permissions.
Use git clone instead of ZIP download — if you have Termux or a code editor app on Android with git support, you can clone the repo directly using the Replit git remote URL (Tools → Git in the workspace).
Download from a laptop/desktop if available — this is the most reliable path since ZIP downloads are primarily built for desktop browsers.
As for "is there a new update" — I don't have visibility into Replit's release notes or upcoming platform fixes for this. If it's a persistent bug rather than a one-off, I'd recommend reaching out to Replit support directly with the specific error message you're seeing (a screenshot helps a lot), since this is a platform-level issue outside what I can change in your project.
