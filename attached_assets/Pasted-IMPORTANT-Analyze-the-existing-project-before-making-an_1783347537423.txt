CUSTOMER IDENTIFICATION

Do NOT require customer login.

Do NOT require registration.

Do NOT require OTP.

Do NOT require Firebase Authentication.

Instead use:

• Order ID
• Mobile Number

for secure order tracking.

==================================================

ORDER ID

Whenever a customer submits a medicine request successfully:

Generate a unique Order ID.

Example:

AYM-2026-000123

After submission show:

✅ Request Submitted Successfully

Your Order ID:

AYM-2026-000123

Please save this Order ID.

You can use it later to track your medicine request.

==================================================

TRACK ORDER

Track Order should ask for:

• Order ID

• Mobile Number

Only if BOTH match a Firestore document should the order be displayed.

Otherwise show:

Order not found.

Please verify your Order ID and Mobile Number.

Never reveal whether:

• the Order ID exists

or

• the mobile number is incorrect.
